# Прогнозирование спроса на такси

## Описание проекта
Цель проекта — разработать модель машинного обучения для прогнозирования количества заказов такси по часам. Решение направлено на оптимизацию распределения водителей, снижение времени ожидания клиентов и повышение операционной эффективности сервиса.

Модель строится на основе исторических данных о заказах такси в аэропортах и использует методы анализа временных рядов и машинного обучения.

---

## Навыки и инструменты
* pandas  
* NumPy  
* matplotlib, seaborn  
* Анализ временных рядов  
* Feature Engineering (lag features, rolling mean)  
* Scikit-learn  
* CatBoost / Gradient Boosting / Random Forest  
* Метрики качества: RMSE  

---

## Данные
Использовались исторические данные заказов такси:

* Временная метка заказа  
* Количество заказов в час  
* Период наблюдений — несколько месяцев  

---

## Этапы работы

### 1. Предобработка данных
* Проверка пропусков  
* Сортировка по времени  
* Ресемплирование данных по часам  

---

### 2. Исследовательский анализ данных (EDA)
* Построение временных рядов  
* Анализ тренда и сезонности  
* Анализ распределения заказов  
* Анализ спроса по времени суток  

---

### 3. Подготовка признаков
Были созданы дополнительные признаки:

* Час  
* День недели  
* Лаговые признаки  
* Скользящие средние  

---

### 4. Обучение моделей
Были протестированы несколько моделей:

* Linear Regression  
* Random Forest  
* Gradient Boosting  
* CatBoost  

---

### 5. Выбор лучшей модели
* Основная метрика — RMSE  
* Требование бизнеса — RMSE ≤ 48  
* Лучшая модель выбрана на основе результатов на тестовой выборке  

---

## Итоговые выводы

В ходе анализа было выявлено, что спрос на такси имеет выраженную сезонность и зависит от времени суток и дня недели. Пики заказов наблюдаются в часы повышенной транспортной активности.

Добавление временных признаков и лаговых значений позволило значительно повысить качество прогнозирования.

Разработанная модель показывает высокую точность прогнозирования и соответствует бизнес-требованиям проекта. Решение может использоваться для:

* планирования загрузки водителей  
* оптимизации распределения автомобилей  
* снижения времени ожидания клиентов  
* повышения качества сервиса  

Использование модели позволяет:

* снизить операционные затраты  
* повысить удовлетворенность клиентов  
* улучшить планирование ресурсов  
* принимать решения на основе данных
